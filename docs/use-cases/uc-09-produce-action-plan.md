# UC-09: Produce risks and an executable action plan

**Milestone:** Phase 1C - Analysis and Action  
**Status:** Draft for review

## Goal

Translate bounded legal analysis into practical next steps without inventing local requirements or performing actions on the actor's behalf.

## Main flow

1. The Agent derives risks and options from the approved analysis version.
2. Risks are described by nature, likelihood basis, impact, urgency, warning signs, and mitigation—not by an unsupported single percentage.
3. Options list prerequisites, trade-offs, dependencies, and circumstances in which they cease to apply.
4. The action plan orders steps, identifies the responsible actor, required information or documents, applicable authority, timing basis, and completion evidence.
5. Local fees, forms, offices, and deadlines are included only when verified for the resolved locality and relevant date.
6. Unknown operational details become verification tasks rather than fabricated instructions.
7. Any external submission, booking, communication, or irreversible act requires a separate explicit human confirmation and authorized integration.

## Alternate and failure flows

- Urgent deadlines are highlighted and routed for review when not safely verifiable.
- Unsupported local procedure coverage produces a general preparation plan plus a verification/escalation step.
- Competing options remain separate; the Agent does not choose on behalf of the user when value judgment is required.

## Required records

Analysis dependency; risks; options; ordered actions; actors; prerequisites; timing and authority basis; locality verification; completion evidence; external-action approval state.

## Acceptance criteria

- Each legal or procedural instruction links to supporting analysis and verified source context.
- Steps are ordered, actor-specific, and testably completable.
- Unknown fee, deadline, office, or form is never invented.
- No external action occurs within this use case.
- Material analysis changes mark dependent plans stale.

## Evaluation evidence

Task executability review, local-procedure correctness, fabricated-detail rate, dependency consistency, urgency handling, and external-action guardrail tests.
